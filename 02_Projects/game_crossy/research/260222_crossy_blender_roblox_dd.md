## 1. Blender MCP 현황 (2025-2026)

### 요약 및 결론
**현재(2026 초) Blender MCP만으로 "프롬프트를 통해 바닥부터 완성도 높은 로우폴리 గేమ్ 에셋을 생성"하는 것은 비효율적이다.** 
대신, 외부 AI 3D 생성기(Meshy/Tripo3D 등)로 베이스 모델을 먼저 뽑고, *Blender MCP를 통해 씬 구성, 재질 조작, 리깅 보조, 로블록스 수출 최적화 등을 제어하는 용도*로 쓰는 것이 가장 강력한 조합이다.

### 세부 내용
- **주요 프로젝트:** `ahujasid/blender-mcp`, `CommonSenseMachines/blender-mcp` 등 다수의 실험적/오픈소스 프로젝트가 GitHub에 존재하며 활발히 유지보수 중.
- **작동 방식:** Blender 내부에 소켓 서버 애드온을 설치하고, Python 기반 MCP 서버가 이를 언어 모델(Claude 등)과 중계. AI가 Python 스크립트를 작성하여 모델링, 재질 적용, 씬 렌더링을 제어.
- **핵심 한계:** AI가 Python의 구조적인 명령(Vertices/Faces 조작)으로 기하학적으로 완벽한 로우폴리 에셋을 바로 "조립"하는 것은 LLM의 공간/시각 지능의 한계로 인해 프로덕션 레벨이 아님. 하지만 **AI 3D 파운데이션 모델(Rodin 등)과의 API 연동**을 스크립트로 처리하거나, 불러온 에셋을 조작하는 데에는 매우 유용함.

## 2. Roblox 3D 에셋 파이프라인

### 요약 및 결론
**Blender에서 생성한 에셋을 Roblox로 넘기는 작업은 FBX 임포트 포맷을 통해 고도로 표준화되어 있다.** 단, 단일 메쉬 2만 폴리곤, 텍스처 해상도 최대 2K(모바일 1K)라는 한계를 반드시 지켜야 한다.

### 세부 내용
- **임포트 워크플로우:** `.FBX` 포맷 권장. OBJ와 달리 계층 구조, 텍스처/재질 설정, 리깅 정보를 가장 잘 보존함.
- **스펙 제한 (2026년 기준):**
  - **최대 폴리곤:** **단일 MeshPart 당 20,000 Tris** (과거 1만 개에서 상향됨). 단, 모바일/대규모 게임 최적화를 위해서는 에셋당 **3,000 ~ 4,000 Tris** 내외의 로우폴리 권장. CSG(솔리드 지오메트리)는 5,000.
  - **텍스처:** 알베도, 러프니스, PBR 맵 기준 최대 **2048x2048 (2K)** 업로드 지원. 단, 모바일 기기(메모리 3.5GB 이하)는 1024x1024 (1K)로 자동 다운스케일 렌더링.
  - **블렌더 최적화 팁:** 작업 시엔 4K로 작업해도 무방하나, 로블록스용 PBR 텍스처를 베이킹할 땐 1K~2K로 뽑고 N-Gon(5각형 이상) 없는 사각/삼각 폴리곤(Quads/Tris)만 사용할 것.

## 3. Roblox 게임 에셋 시장 트렌드

### 요약 및 결론
**로우폴리, 카툰 렌더링화된 스타일라이즈드(Stylized) 에셋은 여전히 Roblox의 최대 주류(트렌드)이며 앞으로도 변함없다.** AI로 대량 생산하기 가장 이상적인 타겟 시장이다.

### 세부 내용
- **트렌드:** Adopt Me, Tower Defense 류의 수십억 플레이 히트작들은 모두 일관되게 '로우폴리, 밝은 색감, 단순화된 형태'를 사용.
- **이유:** 낮은 사양의 폰, 태블릿 접속 스펙을 커버하기 위함 + 로블록스 특유의 블록형 아바타와 융화되는 심미적 타겟층(Gen Z, Alpha)의 취향.
- 실사풍(Photorealistic) 에셋도 시도되나 최상위 매출 게임의 90% 이상은 로우폴리 텍스처의 스타일라이즈드 지형과 프롭을 채택함.

## 4. 경쟁 도구/대안 비교 (AI 3D vs 전용 툴)

### 요약 및 결론
Roblox 개발 1인 파이프라인으로는 **Meshy(초보자/프로토타이핑/최고속도)** 또는 **Tripo3D(최적화/쿼드메쉬/게임레디 품질)**의 압승이다. Blender 내장 스크립팅만 고집할 필요가 없다.

### 세부 내용
| 도구 | 특징 및 Roblox 호환성 | 가격/비용 | 추천 대상 |
|---|---|---|---|
| **Meshy** | 초고속(약 45초). "Roblox" 타겟 프롬프팅 최적화. UI 매우 쉬움. | 월 정액제 (무료 크레딧 제공) | **초기 기획자 / 빠른 그레이박스 대체** |
| **Tripo3D** | 게임 엔진용 토폴로지 우수(사각 메쉬). "Smart Retopology"로 폴리곤 1만 개 미만 최적화 기능 내장. | 월 구독 / 크레딧 | **실제 Roblox 배포용 품질 에셋** |
| **Rodin (Hyper3D)** | 극상의 퀄리티, 실사풍 4K PBR 특화. 렌더 속도 느림. 폴리 수도 높음. | 세대 당 약 $0.75 | 하이엔드 UGC 에셋 판매자. 일반 카툰 게임엔 오버킬. |
| **Blender MCP 단독** | Python 코드 기반 쌩(?) 제너레이션. 메쉬 기하학 에러 빈번. | 무료 기반 (API 비용만) | **생성보다 씬 "제어/배치/리깅" 보조용** |

**Workflow 제안:** `프롬프트 -> Tripo3D (1차 로우폴리 메쉬/텍스처 생성) -> Blender MCP (로블록스 호환 스케일 조정, 리깅 확인, FBX Export 자동화) -> Roblox Studio`

## 5. 1인 개발자 관점의 실용성 평가

### 요약 및 결론
**"그레이박스 프로토 → 재미 검증 → AI 에셋(Meshy/Tripo3D) 교체" 전략은 현재 1인 개발자 메타에서 가장 빠르고, 파괴적이며, 완벽히 실현 가능한 파이프라인이다.**

### 세부 내용
- **학습 곡선 예상:** 개발자가 Blender 단축키나 모델링 스컬핑(Sculpting)을 1도 몰라도, Mesh와 Tripo3D에서 생성된 결과물을 Blender(MCP 어시스턴트의 도움)에 띄워 포맷 자동 변환만 지시하면 끝난다.
- **엔드-투-엔드(End-to-End) 현실성:** 매우 높음. 텍스처 베이킹, 메쉬 병합, FBX 익스포트 세팅 같은 블렌더 특유의 번거로운 작업들을 MCP(AI 챗)가 대신 스크립트로 처리해 줄 수 있으므로 생산성에 날개를 달 수 있음.
- **Godot 대비 압도적 이점:** 1인 개발자가 싱글/멀티플레이 서버, 인벤토리 물리 로직, 결제 시스템(Robux) 연결까지 직접 다 짜야 하는 Godot 보다 "그레이박스" 제작이 훨씬 빠름. 어차피 AI로 에셋을 바를 거라면 Roblox 플랫폼에 얹는 게 TTM(Time-To-Market) 관점에서 압도적으로 유리.

## 🚀 최종 권고안 (Recommendation)
### 결정: **PIVOT (Roblox 직행 및 AI 3D 에셋 파이프라인 채택)**

**핵심 근거 요약:**
1. **기술 성숙도:** Tripo3D/Meshy의 현재 품질이 2만 폴리 이하의 로블록스 카툰 에셋 수준을 충분히 상회함.
2. **시장 트렌드 부합성:** 로우폴리 카툰 디자인은 로블록스의 최대 수익 시장임. AI로 가장 쉽게 뽑아내면서도 거부감이 없는 형태임.
3. **Blender MCP의 올바른 활용성:** AI에게 '모델링'을 시키는 대신, AI 3D 툴 결과를 받아 '로블록스 임포트 파이프라인 최적화(단위 변경, FBX 굽기)'를 담당하는 백엔드 일꾼으로 삼으면 막힘없는 End-to-End가 가능하다.
4. **리소크/초기 투입 절감:** Crossy 같은 아케이드/캐주얼 장르는 무조건 멀티/크로스플레이 모수가 핵심. Godot으로 인프라 바닥부터 짜는 것보다 Roblox Studio + AI 에셋 교체 전략의 생존률이 월등히 높음.
