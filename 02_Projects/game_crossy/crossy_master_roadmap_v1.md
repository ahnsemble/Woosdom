## 전략 결정: PIVOT — Roblox 직행

**3자 컨센서스:** Godot 자체 게임 개발 → Roblox 플랫폼 직행 + AI 3D 에셋 파이프라인

**핵심 근거:**
1. Roblox는 서버/결제/매칭 인프라가 제로비용 — 1인 개발자 TTM 압도적 유리
2. 로우폴리 스타일라이즈드 = Roblox 상위 매출 90% + AI 생성 최적 타겟
3. Tripo3D/Meshy의 현재 품질이 Roblox 2만 폴리 규격을 충분히 충족
4. Blender MCP는 "생성"이 아닌 "FBX 변환/스케일 보정" 파이프라인 자동화로 활용

---

## 에셋 파이프라인

```
프롬프트 → Tripo3D/Meshy (로우폴리 메쉬+텍스처 생성, ~45초)
         → Blender MCP (로블록스 스케일 보정, FBX 자동 Export)
         → Roblox Studio (MeshPart 임포트)
```

**Roblox 에셋 규격:**
- 단일 MeshPart: 최대 20,000 Tris (권장 3,000~4,000)
- 텍스처: 최대 2K (모바일 자동 1K 다운스케일)
- 포맷: FBX 권장 (계층구조/재질/리깅 보존)

## 12주 로드맵

### Phase 1: Godot 재미 검증 (Week 1-2)

| Week | 목표 | 산출물 | Go/No-Go |
|------|------|--------|----------|
| **Week 1** (2/24~3/1) | 1분 런 풀루프 MVP | 60초 런 시작→종료→즉시 재시작, 3택 스킬 선택 UI, 세그먼트 6~8개, 스킬 4개 | 60초 런이 10회 연속 끊김 없이 플레이 가능 |
| **Week 2** (3/2~3/8) | 재미 Y/N 판정 | 스킬 8~10개, 세그먼트 10~14개, 난이도 커브, 텔레메트리 | 아래 정량 기준 충족 |

**구현 범위 (여기서 멈춘다):**
- 1분 런 루프: 시작 → 60초 생존/통과 → 결과 → 1초 내 재시작
- 3택 스킬: 매 런 2~3회 선택 (0초/20초/40초)
- 스킬 8~10개: 대시, 더블점프, 슬로우, 무적 0.5s, 트랩 감지 등 체감 큰 것 위주
- 절차 생성: 타일/세그먼트 프리팹 조합 (단순 이어붙이기)
- 텔레메트리: 로컬 로그 (런 수, 생존시간, 스킬 선택 분포, 사망 원인 Top3)

**명시적 제외:** 영구 메타 저장, 상점/결제, 아트 고도화, 멀티플레이

**재미 Y/N 정량 판정 기준 (모두 충족 시 Go):**

| 구분 | 지표 | 기준 |
|------|------|------|
| 자체 플레이 | 연속 12분(8~12런) 의무감 없이 유지 | Y/N |
| 자체 플레이 | 런 종료 → 재시작까지 평균 3초 이하 | ≤ 3초 |
| 외부 테스터 | 테스터 8명+, 각 15분+ 플레이 | — |
| 외부 테스터 | 48시간 내 자발적 재접속/의향 | ≥ 5/8명 |
| 외부 테스터 | "스킬 선택이 재미를 만든다" 동의 | ≥ 3.8/5.0 |
| 행동 데이터 | 최초 15분 1인당 평균 런 시도 | ≥ 6회 |
| 행동 데이터 | 스킬 선택 화면 이탈률 | ≤ 15% |

**No-Go 시:** 1회 콘셉트 수정 후 재평가 → 재차 미달이면 프로젝트 보류 (매몰비용 2주)

**Godot에서 가져갈 것 vs 버릴 것:**

| 가져갈 것 (문서/데이터) | 버릴 것 (코드/구현체) |
|------------------------|---------------------|
| 스킬 풀 정의서 (효과/쿨/상호작용) | GDScript 코드 전체 |
| 세그먼트 카탈로그 (소요시간/시너지) | Godot 씬 구조 |
| 난이도 커브 가설 | 물리/충돌 구현체 |
| 텔레메트리 이벤트 설계 | 절차 생성 코드 (개념만 유지) |

**기술 스택:**
- **프레임워크:** Knit 패턴만 차용 (Service/Controller), 핵심 로직은 순수 모듈 (Knit archived 상태이므로 의존도 최소화)
- **데이터:** ProfileStore (세션 락, 자동 저장)
- **이벤트:** GoodSignal (task 기반, 메모리 누수 회피)
- **빌드:** Rojo + Wally 패키지 매니저

**서버-클라 분리 원칙:**

| 서버 (권한/보안) | 클라이언트 (UX/연출) |
|-----------------|-------------------|
| 런 시작/종료, 체크포인트 | UI 전부 (스킬 선택, 상점, 결과) |
| 사망 판정, 보상 지급 | 카메라, 이펙트, 사운드 |
| RNG: 스킬 3택 롤링 | 로컬 입력 (결과는 서버 동기화) |
| 절차 맵 생성 (시드) | 스킬 선택 팝업 애니메이션 |
| 결제 영수증 처리 | — |

**절차적 맵 생성:**
- 세그먼트(Model 프리팹) Start/End Attachment 통일
- `Random.new(seed)`로 재현 가능 시퀀스
- 난이도 티어/금지 조합/반복 제한 룰
- 플레이 진행에 따라 앞쪽 N개만 유지, 뒤쪽 정리
- 난이도 커브: 0~20초 쉬움 → 20~40초 중간 → 40~60초 어려움
