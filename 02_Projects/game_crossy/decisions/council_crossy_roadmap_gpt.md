# Project Crossy 세부 실행 로드맵

## 핵심 답변

아래 로드맵은 **주당 약 15–18시간(평일 7–10h + 주말 8–12h)**의 제약을 전제로, “Godot로 1분 런 재미 검증 → Luau로 재설계/재개발 → Roblox 베타 → 정식 출시”의 **리스크 최소 경로**로 설계했습니다. (Week 표기에는 실제 날짜 범위를 함께 적었습니다.)

### Godot 미니멀 프로토 스코프

**정확히 여기까지 구현하고 멈춥니다(“포팅”이 아니라 ‘재개발’이니 코드 재사용을 목표로 하지 않음).**

**구현 범위(필수)**
- 1분 런 루프: 시작 → 60초 생존/통과 → 결과 → 1초 내 재시작(“One more run” 압축).
- 3택 스킬: 매 런 2~3번(예: 0초/20초/40초)에 “3개 중 1개 선택”. 선택은 완전 랜덤(가중치 없이 시작).
- 스킬 8~10개(초기 확정 범위): “체감이 큰 것” 위주(대시/더블점프/슬로우/무적 0.5s/트랩 감지 등).
- 절차 생성 트랩 구간: **‘타일/세그먼트 조합’**(프리팹 N개 중 랜덤으로 이어붙이기). 알고리즘은 단순해야 합니다(Week2 안에 완성되어야 하므로).
- 최소 텔레메트리: 로컬 로그라도 좋음(런 횟수, 평균 생존시간, 스킬 선택 분포, 사망 원인 Top3).

**멈추는 지점(명시적 제외)**
- 영구 메타 저장(데이터 영속성), 라이브옵스, 상점/결제, 아트 고도화, 멀티플레이/코플레이.
- “Roblox에서 필요한 최적화/네트워킹/치팅 방어”도 Godot에서는 하지 않습니다.

#### “재미 Y/N” 정량 판정 기준
다음 기준을 **모두** 만족하면 “Y”, 하나라도 크게 미달하면 “No-Go(혹은 1회 수정 후 재평가)”로 둡니다.

**자체 플레이 기준(당신 1인)**
- **연속 12분(=대략 8~12런)** 플레이를 “의무감 없이” 유지 가능(중간에 억지로 참는 느낌이 들면 실패).
- 1분 런 종료 직후, **재시작까지 평균 3초 이하**로 눌러지는지(UX/리듬 체크).

**외부 테스터 기준(최소 표본)**
- 테스터 **8명 이상**, 각자 **최소 15분 플레이**.
- 그중 **5명 이상**이 “48시간 내 자발적 재접속(또는 다시 해보고 싶다 응답)”을 남김.
- “스킬 선택이 재미를 만든다”에 동의(5점 척도 평균 3.8+/5).  
  - 여기서 중요한 건 밸런스가 아니라 **‘선택이 도파민을 준다’**는 체감입니다.

**행동 데이터 기준(텔레메트리)**
- 최초 15분 동안 1인당 평균 런 시도 **≥ 6회**(실패해도 다시 누르는가).
- 스킬 선택 화면에서 이탈(선택 안 하고 종료) 비율 **≤ 15%**.

#### 프로토에서 Roblox로 가져갈 것 vs 버릴 것

**가져갈 것(문서/데이터/경험)**
- 스킬 풀(8~10개) 정의서: 효과/쿨타임/상호작용/희귀도(가중치) 초안.
- 구간 프리팹 카탈로그: “이 프리팹은 평균 몇 초 걸리고, 어떤 스킬과 시너지가 있나” 같은 운영 데이터.
- 난이도 커브 가설(초반 진입 30초는 ‘무조건 재미’, 후반에서 실패 유도).
- 텔레메트리 이벤트 설계(런 시작/종료, 사망 원인, 스킬 픽).

**버릴 것(코드/구현체)**
- Godot 이동/점프/물리 구현체(완전 폐기).
- Godot 씬 구조(아이디어만 참고).
- 절차 생성 코드(‘알고리즘 개념’만 유지, Luau로 재작성).

### Luau 전환 기술 로드맵

#### Knit/ProfileStore/GoodSignal 세팅 순서(권장)

1) **프로젝트 구조 먼저 고정**  
- ServerScriptService / ReplicatedStorage / StarterPlayerScripts 기준으로 “서버 권한 로직 vs 클라 UI/연출”을 분리합니다. (Week3 DoD)

2) **Signal 레이어(이벤트 버스) 확정**  
- GoodSignal은 “BindableEvent 기반이 아니라 task 기반의 순수 Lua 구현”으로 메모리 누수 회피를 강조합니다. citeturn1search1  
- 또는 RbxUtil Signal은 GoodSignal을 사실상 복제/확장했다는 설명이 있어(타이핑/추가 메서드) 대안이 됩니다. citeturn1search17  
→ 1인 개발이라면 “어느 Signal을 쓰든 **한 번 정하고 통일**”이 핵심입니다.

3) **ProfileStore로 영구 데이터 고정**  
- ProfileStore는 DataStore 래퍼로 “자동 저장, 세션 락(session locking)” 등을 전면에 내세웁니다. citeturn1search0turn1search4  
- Roblox 공식도 “Data Stores 최적화/베스트 프랙티스”를 별도 문서로 제공합니다(쿼터/버전/최적화 등). citeturn1search33  

4) **(선택) Knit 도입 여부 최종 결정**  
- Knit 저장소는 “archived / no longer maintained”로 명시돼 있습니다. citeturn0search3  
- 즉, Knit을 쓰더라도 “프레임워크를 믿는 게 아니라, **Service/Controller 패턴을 빌려 쓰는 것**”에 가깝게 접근해야 리스크가 줄어듭니다.  
(이 부분은 아래 리스크 매트릭스에도 반영)

## 근거

Roblox의 **추천/디스커버리 신호**는 공개 문서·공지에서 일부 정의가 노출되어 있으며, 특히 Home 추천에서 **qPTR(qualified play-through rate), 7일 플레이타임/플레이데이, 7일 지출 관련 지표, 7일 코플레이 지표**가 언급됩니다. citeturn5search4turn12search2turn5search20  
또한 7일 플레이타임은 “유저당 일일 60분 캡”이 언급되어, “짧고 반복 가능한 세션”이 유리한 해석이 가능합니다. citeturn5search4turn5search36  

Creator Rewards는 “10분 기준 + 그날 첫 3개 경험 + Active Spender 조건 + 60일 홀드”라는 구조를 명시하고, 10분을 넘겨도 추가 보상이 없다고 밝힙니다. citeturn14search8turn14search14  
따라서 1분 런 게임은 “런을 많이 하게 만들기”만으로는 부족하고, **10분 체류를 만들 수 있는 데일리/코플레이/루프 확장 장치**가 필요합니다.

Ads Manager는 자동 입찰/썸네일 다변량 테스트(최대 10개, 균등 분배), Home+Search 노출, CTR/CPP 등 지표 정의, 로벅스→애드크레딧 전환비(263 Robux)까지 문서화되어 있어, 소액으로도 “크리에이티브 실험→승자 확대”가 가능합니다. citeturn11view0turn9view0  

Moments 생태계는 Roblox가 직접 “Obby에서 자동 캡처→공유→하이라이트 릴”을 예시로 들며, Share Link API로 딥링크/재참여를 지원합니다. citeturn4view3turn16view0turn15view0  
다만 동시에 Maturity/Compliance 업데이트 같은 운영·정책 부담이 강조되어, 1인 개발에서는 “최소 통합→안정화 후 확장”이 더 안전합니다. citeturn4view3turn16view0  

추가로, 2025-12-17부터 “퍼블릭 경험 게시/업데이트”에 ID 인증 또는 일정 기준의 유료 결제 이력이 요구된다는 공지가 있어, 1인 개발자의 “배포 가능성” 자체가 정책 리스크가 됩니다. citeturn17view0

## 가정과 한계

- 이 로드맵은 “Obby × Roguelike”가 실제로 시장에서 먹히는지 **Godot 2주 검증으로 반드시 판정**한다는 전제입니다. (Y가 아니면 Roblox 재개발은 비용 대비 리스크가 큼)

- UGC는 계정 요건/심사/정책 변화가 잦고, 직접 제작보다 협업이 현실적인 경우가 많습니다. Marketplace 정책의 요건(예: ID/Premium)은 반드시 최신 문서로 재확인해야 합니다. citeturn3search11turn3search15
