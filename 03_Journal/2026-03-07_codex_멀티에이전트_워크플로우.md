# Codex 멀티에이전트 워크플로우 대화 기록
**날짜:** 2026-03-07
**주제:** 멀티에이전트 워크플로우와 Codex 적용 방식

---

## 1. 멀티에이전트 워크플로우란?

한 개의 AI가 모든 일을 처리하는 대신, 역할이 다른 여러 에이전트가 분업해서 하나의 작업을 끝내는 방식이다.

예시 역할:
- `Planner`가 목표를 분해
- `Researcher`가 자료 조사
- `Executor`가 실제 실행
- `Reviewer`가 결과 검토

핵심은 단순히 에이전트 수가 많은 것이 아니라, 아래가 정의되어 있어야 한다는 점이다.
- 누가 어떤 순서로 일하는지
- 어떤 기준으로 다음 단계로 넘기는지
- 어떤 결과물을 다음 에이전트에 전달하는지

Woosdom 기준으로는 `Brain(전략) -> Hands(실행) -> Memory(기록)` 구조도 멀티에이전트 워크플로우의 한 형태로 볼 수 있다.

---

## 2. Codex에서 어떻게 적용하나?

질문:
> codex에서 어떻게 적용해? config.toml에 역할정의하면 되는거야?

답변 요지:
- 맞지만, 정확히는 `profiles`가 아니라 `agents`를 정의하는 것이 핵심
- `config.toml`은 전역 실행 설정과 멀티에이전트 역할 등록에 사용
- 역할 자체는 `[agents.<name>]` 아래에 둠
- 프로젝트 규칙은 `AGENTS.md`에 둠

즉 역할 분담은 다음처럼 나뉜다.
- `.codex/config.toml`의 `[agents]`: 역할 정의
- 각 역할의 `config_file`: 역할별 모델/샌드박스/지시문
- `AGENTS.md`: 저장소 또는 프로젝트 단위 규칙
- `profiles`: 세션 프리셋, 역할 정의용이 아님

---

## 3. Codex 설정 구조 정리

### `config.toml`
- 멀티에이전트 기능 활성화
- 에이전트 목록 및 실행 한도 정의
- MCP 서버, 모델, 샌드박스, approval 정책 같은 실행 설정 관리

예시:

```toml
[features]
multi_agent = true

[agents]
max_threads = 4
max_depth = 1

[agents.explorer]
description = "Read-only code explorer"
config_file = "agents/explorer.toml"

[agents.reviewer]
description = "Find bugs, regressions, and missing tests"
config_file = "agents/reviewer.toml"
```

### 역할별 설정 파일

예시:

```toml
model = "gpt-5.4"
sandbox_mode = "read-only"
developer_instructions = """
코드 경로를 추적하고 증거만 수집해라.
수정안은 요청받을 때만 제안해라.
"""
```

### `AGENTS.md`
- 프로젝트 단위 운영 규칙
- 파일 구조, 수정 금지 대상, 도메인별 제약 등
- Codex가 작업 전 컨텍스트로 읽는 지침 파일

이 볼트에서는 이미 루트 `AGENTS.md`가 존재하며, Obsidian 링크 보존, 템플릿 직접 수정 금지, 금융 데이터 연산 위임 금지 같은 규칙이 정의되어 있다.

---

## 4. 핵심 결론

1. Codex 멀티에이전트는 `config.toml`에서 역할을 등록해 적용한다.
2. 하지만 역할 정의는 `profiles`가 아니라 `[agents]`에 둬야 한다.
3. `AGENTS.md`는 역할 목록이 아니라 프로젝트 규칙과 작업 제약을 담는 파일이다.
4. 실전 적용은 `config.toml` + 역할별 설정 파일 + `AGENTS.md`의 조합으로 보는 것이 맞다.

---

## 5. 짧은 실행 메모

Woosdom에 바로 대응시키려면 다음 식으로 매핑하면 된다.
- `brain-planner`: 작업 분해와 우선순위화
- `code-explorer`: 읽기 전용 탐색
- `executor`: 실제 파일 수정 및 실행
- `reviewer`: 회귀와 누락 테스트 점검
- `memory-keeper`: 결과를 볼트 문서로 정리

이때 프로젝트 공통 규칙은 루트 `AGENTS.md`에, 역할별 성격 차이는 `.codex/config.toml`과 각 role config 파일에 두는 구성이 가장 자연스럽다.
